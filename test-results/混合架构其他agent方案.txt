1. ROLE（角色定位）— 1句话
你是Lexar存储品牌专业翻译，专精消费电子/数码3C/存储行业各种翻译。

2. IRON RULES（全局铁则）— 唯一规则源
1. 品牌名/系列名禁止翻译 (Silver/Blue/Gold/Diamond 是系列名非颜色)
2. 技术规格/行业标准符号原样保留
3. 品类词固定区分 (Desktop Memory ≠ Laptop Memory ≠ SSD ≠ Portable SSD...)
4. 术语库译法最高优先级，数字/容量/速度值原样
5. 读取速度≠写入速度，严格区分
6. 禁止在括号添加解释/原文/别名 → 纯译文输出
7. 短标签(<15字符)禁止扩写成句
8. 忠实原文：不漏译、不增译、不自行补充卖点


3. STYLE（翻译风格）— 1行

  注入条件: 仅电商场景下注入（技术参数/包装/UI/售后/说明书/规格书场景跳过）
  触发逻辑:
    ┌─ 用户手动选择: standard / professional / marketing / custom
    ├─ 非电商场景(technical_params/packaging/ui/after_sales/manual/spec_sheet):
    │   └─ 强制锁定为 professional，且跳过独立风格块（场景已内置）
    └─ 电商场景: 注入选中的风格

  注入内容:
    standard     → "自然流畅，句式通顺，符合目标语言表达习惯"
    professional → "严谨正式，技术表述精准客观。句式简洁"
    marketing    → "准确传达卖点，句式有吸引力和说服力。保持高端品牌质感"

技术参数/包装/UI/售后/说明书/规格书场景：分别有不同的提示词注入。




我的大概想法是这个注入逻辑
  │  ① ROLE                    ── 1句，始终注入（人设）                              │
  │  ② GLOBAL_RULES            ── 8条铁则，始终注入（品牌翻译规则）                          │
  │  ③ TRANSLATE_TASK          ── 翻译方向声明，始终注入（具体由什么语言翻译成什么语言，详细的任务）                      │
  │  ④ TRANSLATION_CONTEXT     ── 三合一上下文块，始终注入（永不跳过）         │
  │  ⑤ CATEGORY_WORDS          ── 品类词对照表，按产品线过滤                  │
  │  ⑥ GLOSSARY                ── 术语库注入，三层过滤                        │
  │  ⑦ CONTEXT_HINT            ── 跨条目语义连贯提示                          │
  │  ⑧ LANG_SPECIFIC           ── 20语种专属规则，目标语言匹配时注入           │
  │  ⑨ OUTPUT_ANCHOR           ── 输出纯度锚点（末尾，Qwen注意力最高区）       │
  │  ⑩ MARKERS                 ── __XXX_N__ 占位符保护                        │
  │  ⑪ FORMAT + few-shot       ── 输出格式 + 翻译参考示例     

我们设计的目的是为了提升翻译质量，减少token消耗，让提示词更加贴合我们具体的产品线与目标受众，颗粒度足够的细致，就越具有增对性。


1.只有商品详情页会有三种不同的翻译风格，其他场景都是严谨专业的感觉。2.请帮我根据实际情况把所有的提示词补充完整。3.考虑各种使用场景，以及每个模块所起到的作用。模拟测试各种使用场景比如英文翻译中文，中文翻译繁体中文，英文翻译成日语，意大利语，阿拉伯语，越南语等，以及不同的使用场景，产品线是电竞SSD，是PC生产力等，详情页不同的翻译风格，他们最终的提示词是怎样组合的，有没有相互污染，有没有重复冗余，怎么才能让提示词更加高效，颗粒度更细，更有针对性。 3.我们混合架构的设计目的是为了提高翻译质量，不是为了创作，让我们的提示词更加贴合实际的产品线与目标客群，翻译的词语调性更加符合我们的产品与目标人群。 4.可以用网络搜索我们各个产品线的文案风格，有针对性的注入提示词来提升翻译质量 5.术语库我们是通过确认产品线然后通过这个产品线专属的标签以及通用标签的形式来注入以减少token的数量。   6.你在虚拟一些不同的产品线，来测试是否重复污染冗余吧，目的是提示词颗粒度更加的细致，符合具体的产品线以及人群的语感与调性，拉近与消费者的具体，也是为了去除机翻感。7.有个补充的信息，大模型我主要使用最新的Qwen3.7模型，可以有增对性进行优化

2. 占位符保护工程化处理（防破坏核心逻辑）
不要指望大模型能完美识别 {username} 或 <br>。

import re

def protect_markers(text):
    # 1. 提取所有需要保护的变量/标签
    markers = re.findall(r'(\{.*?\}|<.*?>|__.*?__)', text)
    # 2. 替换为无意义的唯一占位符
    protected_text = text
    marker_map = {}
    for i, m in enumerate(markers):
        placeholder = f"[[LEXAR_PH_{i:03d}]]"
        protected_text = protected_text.replace(m, placeholder, 1)
        marker_map[placeholder] = m
    return protected_text, marker_map

def restore_markers(translated_text, marker_map):
    # 翻译完成后，在代码层还原
    for placeholder, original in marker_map.items():
        translated_text = translated_text.replace(placeholder, original)
    return translated_text
将 marker_map.keys() 作为列表注入到 Prompt 的 <markers> 模块中。
